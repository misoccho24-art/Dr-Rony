# ব্লগে নতুন পোস্ট দেওয়ার নিয়ম

আপনার ওয়েবসাইটে একটি বিশেষ পাতা আছে যেখান থেকে আপনি সরাসরি ব্লগ পোস্ট
করতে পারবেন — GitHub-এ ঢোকা বা কোনো ফাইল এডিট করার দরকার নেই।

**পাতাটির ঠিকানা:** `আপনার-ওয়েবসাইট.com/admin.html`
(এটি বুকমার্ক করে রাখুন)

## প্রথমবার ব্যবহারের আগে (ডেভেলপার করে দেবে)

এই পাতাটি কাজ করার জন্য একটি "অ্যাক্সেস কোড" লাগবে (GitHub Access
Token)। এটি একবার তৈরি করে আপনাকে দেওয়া হবে — এটি একটি পাসওয়ার্ডের
মতোই কাজ করে, শুধু আপনার এই একটি ওয়েবসাইটে পোস্ট করার অনুমতি দেয়।

## পোস্ট করবেন যেভাবে

1. `admin.html` পাতাটি খুলুন।
2. প্রথমবার আপনার অ্যাক্সেস কোডটি বসিয়ে "লগইন করুন" চাপুন। "এই ডিভাইসে
   মনে রাখুন" টিক দেওয়া থাকলে পরের বার আর কোড দিতে হবে না।
3. শিরোনাম লিখুন।
4. তারিখ (এমনিতেই আজকের তারিখ বসানো থাকবে, চাইলে বদলাতে পারেন)।
5. ছবি দিতে চাইলে "ছবি" অংশে ক্লিক করে আপনার ফোন বা কম্পিউটার থেকে ছবি
   বেছে নিন। ছবি না দিলেও চলবে।
6. নিচে বড় বাক্সে আপনার লেখা লিখুন। নতুন প্যারাগ্রাফের জন্য মাঝে একটি
   ফাঁকা লাইন রাখুন (এন্টার দুইবার চাপুন)।
7. "প্রকাশ করুন" বাটনে ক্লিক করুন।
8. সবুজ ঠিক চিহ্ন দেখালে পোস্টটি সংরক্ষণ হয়ে গেছে। ওয়েবসাইটে দেখা যেতে
   সাধারণত ১-২ মিনিট সময় লাগে।

## পোস্ট মুছে ফেলতে চাইলে

`admin.html` পাতার নিচের দিকে "আগের পোস্টসমূহ" অংশে সব পোস্টের তালিকা
দেখা যাবে। যেটি মুছতে চান তার পাশে "মুছুন" বাটনে ক্লিক করুন।

## নিরাপত্তা সংক্রান্ত কথা

- অ্যাক্সেস কোডটি কারও সাথে শেয়ার করবেন না — এটি দিয়ে যে কেউ আপনার
  ওয়েবসাইটে পোস্ট করতে পারবে।
- অন্য কারও ডিভাইসে লগইন করলে কাজ শেষে "লগআউট" চেপে বেরিয়ে আসুন।
- কোড হারিয়ে গেলে বা কারও কাছে চলে গেছে মনে হলে ডেভেলপারকে জানান —
  নতুন কোড তৈরি করে পুরনোটি বাতিল করে দেওয়া যায়।

### ডেভেলপারের জন্য নোট (সেটআপ)

`admin.js` ফাইলের একদম উপরে এই তিনটি লাইন পূরণ করতে হবে:

```js
const GITHUB_OWNER  = "your-github-username";
const GITHUB_REPO   = "your-repo-name";
const GITHUB_BRANCH = "main"; // or "master"
```

তারপর GitHub-এ গিয়ে **Settings → Developer settings → Fine-grained
tokens → Generate new token** থেকে একটি টোকেন তৈরি করুন:
- Repository access: শুধু এই একটি রিপোজিটরি বেছে নিন।
- Permissions: **Contents → Read and write**।
- Expiration যত দূর সম্ভব বেশি সময়ের জন্য দিন, নাহলে মেয়াদ শেষে
  ক্লায়েন্টকে নতুন কোড দিতে হবে।

এই টোকেনটিই ক্লায়েন্টের "অ্যাক্সেস কোড"। এটি তাকে নিরাপদে (যেমন
WhatsApp-এ সরাসরি, ইমেইলে নয়) পাঠিয়ে দিন।

`blog-posts.txt` ফাইলটি এখনো GitHub-এ প্লেইন টেক্সট হিসেবেই থাকবে,
শুধু এখন সেটি ক্লায়েন্টের হয়ে `admin.html` পাতা থেকেই আপডেট হবে। চাইলে
ডেভেলপার হিসেবে আপনি সরাসরি GitHub-এ গিয়েও ফাইলটি এডিট করতে পারবেন —
ফরম্যাট আগের মতোই আছে (`title:` / `date:` / `image:` / `---` / লেখা /
`===`)।
